# SignalCorps (多角色 AI 互動通訊系統)

一個基於 Google Gemini API 開發的類通訊軟體應用。本專案核心在於探索 **LLM 多角色約束系統**、**動態 API 配額管理** 以及 **自動化對話觸發機制**，展示如何將創新想法透過結構化邏輯落地為可運作的系統模型。

---

## 📌 專案核心亮點與創新思維

本專案不盲目追求複雜的底層框架，而是著重於優化「使用者體驗」與「解決免費 API 的工程限制」：

### 1. 💡 LLM 驅動的多角色衝突系統 (Multi-Agent Simulation)
不同於傳統一對一的 AI 聊天，本專案實作了「群組對話」的動態氛圍：
* **嚴格的角色邊界**：透過精心設計的 System Prompt 約束系統，確保多個獨立角色在同一時空保持各自獨立的說話風格。
* **角色動態互動**：AI 不僅能回應使用者，還能依據上下文做到角色間的互相駁斥與吐槽，形成具有高擬真度的「群組動力學」。
* **結構化輸出約束**：強制 LLM 輸出標準 JSON 格式，包含發言人、延遲時間與訊息內容，以便前端進行精確的 UI 渲染。

### 2. 🔋 獨創的「彈藥系統」：分散式 API Key 輪轉管理
針對 Google Gemini 免費版 API 的每日呼叫限制，本專案設計了一套高容錯的後台調度邏輯：
* **自動輪轉機制**：將多組 API Key 視為「彈藥」，當前 Key 額度耗盡或觸發 Rate Limit 時，系統會自動、無感地切換至下一組。
* **配額智能重置與監控**：實作跨時區的配額重置邏輯，並在介面上提供視覺化的 Key 狀態監控，確保應用的全天候可用性。

### 3. ⏰ 主動式對話觸發機制 (Proactive Engagement)
打破「人問、AI 才答」的被動模式，系統具備後台定時與事件觸發邏輯，能模擬真實通訊軟體中「對方突然主動發起話題」或「隔了一段時間才回覆」的行為，大幅提升互動的沉浸感。

## 🛠️ 技術架構與模組設計

專案採用清晰的層次架構，確保邏輯的解耦與可維護性：

* **UI 互動層**：使用高流暢度的列表元件（RecyclerView）動態渲染多模板訊息（使用者發送 vs 不同 AI 角色的對話氣泡）。
* **AI 核心引擎 (GeminiEngine)**：負責動態構建包含上下文的 System Prompt、呼叫 API，並對 LLM 回傳的資料進行清洗（如移除 Markdown 標記）與防禦性解析。
* **資料持久化層**：整合本地端資料庫（Room），完整儲存對話歷史與 API Key 的狀態，支援完全離線瀏覽。
* **防禦性編程 (Defensive Design)**：針對網路異常、API 限制及未知角色，設計了妥善的 Fallback 機制，確保應用在極端邊界條件下不崩潰。

## 💡 開發心得：人機協同下的軟體開發範式

這個專案是我在工作之餘，作為**「系統架構師與產品經理」**角色，與 **AI 協同開發** 的豐碩成果：
1. **創意思維大於重複代碼**：在現代開發環境中，核心價值在於「如何定義問題」與「設計系統邏輯」。我主導了整體的架構設計、Prompt 工程與防禦性邏輯，並透過 AI 高效完成基礎代碼的編寫。
2. **邊界條件與防禦性設計**：在開發過程中，花費最多心力的並非代碼本身，而是如何引導 AI 去處理 API 逾時、JSON 解析失敗等邊界條件（Edge Cases），這更進一步鍛鍊了我全盤審視系統的工程師素養。
